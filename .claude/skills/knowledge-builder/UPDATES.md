# knowledge-builder 技能更新记录

## 2026-03-19 - 断点续训 & 完整对话存档

### 更新内容

#### 1. 支持中断后继续消化知识（断点续训）

**新增功能**：
- **阶段0：检查训练进度** - 在开始新训练前，先检查是否有未完成的训练进度
- **进度自动保存** - 每完成一个问题，立即保存到 `./knowledge/.progress/[文档名]-progress.json`
- **支持断点续训** - 用户说"继续"、"上次没完成"时，自动从上次停止的地方继续

**新增触发词**：
- "我想继续之前的消化训练"
- "上次没完成，继续"
- "暂停"
- "下次再继续"

**进度文件格式**（JSON）：
```json
{
  "version": "1.0",
  "nourishment_path": "原文档路径",
  "nourishment_title": "文档标题",
  "training_plan": {
    "total_stages": 4,
    "total_questions": 20,
    "stages": [...]
  },
  "progress": {
    "current_stage": 2,
    "current_question": 3,
    "total_completed": 7
  },
  "conversation_history": [
    {
      "stage": 1,
      "question_number": 1,
      "question": "问题内容",
      "user_answer": "用户的完整回答",
      "ai_feedback": "AI的补充/反馈",
      "timestamp": "2026-03-19T10:30:00Z"
    }
  ],
  "last_updated": "2026-03-19T10:30:00Z"
}
```

#### 2. 一问一答完整保存（完整对话存档）

**最终生成两个文件**：

1. **消化理解笔记** - 整理后的核心内容（原有格式）
   - 路径：`./knowledge/digested/digested-[原文件名].md`

2. **完整对话记录** - **一问一答，完整存档，不遗漏任何内容**（新增）
   - 路径：`./knowledge/digested/[原文件名]-full-conversation.md`

**完整对话记录格式**（Markdown）：
```markdown

**类型**：消化训练-完整对话

**开始时间**：YYYY-MM-DD
**完成时间**：YYYY-MM-DD
```

### 新增交互原则

1. **支持断点续训** - 随时可以中断，进度自动保存，下次可继续
2. **完整记录对话** - 每一问一答都完整保存，不遗漏任何内容

### 新增特殊情况处理

**用户说"暂停"、"下次再继续"：**
- 立即保存进度
- 告知用户："好的，进度已保存！下次说'继续'就可以从这里接着来。"

**用户说"继续"、"上次没完成"：**
- 检查进度目录
- 如果有多个未完成进度，列出让用户选择
- 从上次停止的问题继续

### 核心原则（最后强调）

养料消化模式特别记住：
- **进度必存**：每完成一个问题立即保存
- **对话必全**：一问一答完整记录，不遗漏
- **支持续训**：随时可以从中断处继续
