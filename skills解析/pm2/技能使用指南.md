## 什么是技能？

技能（Skill）是结构化的 Markdown 文件（`SKILL.md`），它教会 AI 代理如何专业地完成特定的产品经理任务——每次都用正确的方式，而不需要你从头解释你的流程。

**技能 = 更少的解释，更多的战略工作。**

### 提示词 vs 技能

| 提示词 | 技能 |
|--------|------|
| 每个任务的一次性指令 | 加载一次即可重用的框架 |
| 你需要不断重复自己 | 代理记住最佳实践 |
| "为 X 写一个 PRD" — 听天由命 | 代理知道结构，会问聪明的问题 |
| 每次输出都不一致 | 一致、专业的结果 |

## 三种技能类型

技能按三层架构组织，相互构建。

### 🧱 组件技能（20个）— 模板和工件

**何时使用：** 你需要一个特定交付物的标准模板。
**时间：** 10–30 分钟
**示例：** "写一个用户故事" → `user-story` 技能

**常见组件技能：**
- `user-story` — 写用户故事
- `problem-statement` — 定义问题陈述
- `pol-probe` — 设计验证实验
- `positioning-statement` — 产品定位声明
- `press-release` — 亚马逊逆向工作法的新闻稿

---

### 🔄 交互技能（20个）— 引导式探索

**何时使用：** 你需要在执行前帮助决定采用哪种方法。
**时间：** 30–90 分钟
**示例：** "我应该使用哪个优先级框架？" → `prioritization-advisor` 技能会询问 3–5 个关于你上下文的问题，然后推荐 RICE、ICE、Kano 或其他框架。

**交互技能如何工作：**
1. AI 询问 3–5 个关于你上下文的问题
2. 你回答（或从编号选项中选择）
3. AI 提供 3–5 个量身定制的建议
4. 你选择一个（或组合方法）
5. AI 使用正确的组件技能执行

**常见交互技能：**
- `prioritization-advisor` — 选择优先级框架
- `epic-breakdown-advisor` — 拆分史诗
- `pol-probe-advisor` — 选择验证实验类型
- `context-engineering-advisor` — 上下文工程诊断

---

### 🎭 工作流技能（6个）— 端到端流程

**何时使用：** 你需要从头到尾运行完整的 PM 工作流。
**时间：** 数天到数周
**示例：** "让利益相关者在产品战略上保持一致" → `product-strategy-session` 技能引导你完成定位 → 问题框架 → 解决方案探索 → 路线图。

**常见工作流技能：**
- `prd-development` — 完整的 PRD 创建
- `discovery-process` — 完整的客户发现周期
- `roadmap-planning` — 战略路线图规划
- `product-strategy-session` — 产品战略对齐

## 如何使用技能

### 基本调用模式

无论使用哪个平台，调用技能的基本模式都相同：

```
"使用 [技能名称] 技能，[描述你的任务]"
```

**示例：**
```
"使用用户故事技能，为我们的结账流程编写故事"

"运行优先级顾问，帮助我为 Q3 路线图选择框架"

"对这个假设应用 POL Probe 技能：用户因为运费意外而放弃结账"
```

## 各平台安装指南

### Claude.ai / Claude Desktop
*粘贴、上传或 ZIP — 无需 CLI*

**选项 1 — 直接粘贴技能**
1. 打开任何技能文件夹（例如 `skills/user-story/SKILL.md`）
2. 点击 **Raw** → 全选 → 复制
3. 开始一个 Claude 对话并粘贴：`"阅读这个技能："` 然后粘贴内容
4. 调用：`"使用用户故事技能，为我们的结账流程编写故事"`

**选项 2 — 上传技能 ZIP**
1. 克隆仓库到本地
2. 运行：`./scripts/zip-a-skill.sh --skill user-story`
3. 这会在 `dist/skill-zips/` 中创建一个 ZIP
4. 将 ZIP 文件附加到你的 Claude.ai 消息中
5. 说：`"阅读这个 ZIP 中的 SKILL.md 并将其应用到 [你的任务]"`

---

### Claude Code (CLI)
*原生 CLI — 最简洁的集成*

**设置：**
```bash
# 克隆仓库
git clone https://github.com/deanpeters/Product-Manager-Skills
cd Product-Manager-Skills
```

**从 CLI 调用技能：**
```bash
# 工作流技能
claude "使用 PRD 开发工作流，为我们的移动通知功能创建 PRD"

# 交互技能
claude "运行优先级顾问 — 帮助我为 Q2 排名这 8 个功能"

# 组件技能
claude "使用用户故事技能，为 JIRA-42 中的结账史诗编写故事"
```

**CLAUDE.md — 持久化技能加载**
1. 在项目根目录创建 `CLAUDE.md` 文件
2. 引用你希望始终激活的技能
3. Claude Code 在会话开始时自动读取 `CLAUDE.md`
4. 技能在整个会话中保持加载状态 — 无需重新粘贴

**CLAUDE.md 示例：**
```markdown
# CLAUDE.md

## 活跃的 PM 技能
- skills/user-story/SKILL.md
- skills/prioritization-advisor/SKILL.md
- skills/prd-development/SKILL.md

## 项目上下文
产品：结账重新设计
团队：4 个 PM，2 个设计师，8 个工程师
冲刺：Q2 的史诗分解
```

### ChatGPT / Codex
*GitHub 连接、自定义 GPT 或项目文件*

**选项 1 — GitHub 连接（Codex）**
1. 在 ChatGPT 中，在 **设置 → 集成** 下连接 GitHub
2. 授予 Codex 访问你的 `Product-Manager-Skills` 分支的权限
3. 在 Codex 会话中，通过文件路径引用技能
4. Codex 直接读取仓库 — 无需上传

```
"使用 skills/user-story/SKILL.md，为 docs/checkout-epic.md 中的功能编写用户故事"

"将 skills/prd-development/SKILL.md 应用到这个问题简介：[粘贴简介]"

"运行 skills/prioritization-advisor/SKILL.md 并帮助我排名：[粘贴待办事项]"
```

**选项 2 — 自定义 GPT 知识上传**
1. 前往 **ChatGPT → 我的 GPT → 创建 GPT**
2. 在 **知识** 下，上传你想要的 `SKILL.md` 文件
3. 或者上传技能 ZIP（ChatGPT 自动解压）
4. 在 GPT 指令中，告诉它按名称应用技能
5. 保存并与你的团队分享自定义 GPT

## 脚本工具使用

从本地克隆中使用这些有用的脚本：

```bash
# 按关键词或类型查找技能
./scripts/find-a-skill.sh --keyword pricing --type interactive

# 为一个技能构建 ZIP
./scripts/zip-a-skill.sh --skill user-story

# 为所有技能构建 ZIP
./scripts/zip-a-skill.sh --all --output dist/skill-zips

# 构建精选入门包
./scripts/zip-a-skill.sh --preset core-pm --output dist/skill-zips

# 测试技能的一致性
./scripts/test-a-skill.sh --skill user-story --smoke

# 运行 PM 技能或命令
./scripts/run-pm.sh skill user-story "回头客的结账改进"
```
