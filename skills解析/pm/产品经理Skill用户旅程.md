## 阶段一：战略规划

### 🎯 目标
定义产品愿景、战略方向和商业模式，为后续工作奠定基础。

### 📋 使用技能

| 技能编号 | 技能名称 | 核心输出 | 使用时机 |
|---------|----------|----------|---------|
| **58** | 产品愿景 | 清晰、激励人心的产品愿景 | 项目启动、团队对齐 |
| **57** | 产品策略 | 全面的产品策略 | 战略规划、路线图制定 |
| **51** | 商业模式 | 商业模式分析和设计 | 商业模式设计、收入规划 |
| **52** | 精益画布 | 精益画布可视化 | 创业规划、商业模式验证 |
| **59** | 创业画布 | 创业公司画布 | 创业规划、投资者演示 |
| **50** | 安索夫矩阵 | 增长策略分析 | 考虑增长选项、市场扩张 |
| **54** | PESTLE分析 | 宏观环境分析 | 战略规划、市场进入、风险评估 |
| **55** | 波特五力 | 行业竞争力分析 | 行业分析、竞争评估 |
| **61** | 价值主张 | 价值主张创建和优化 | 产品定位、营销策略 |

### 🔄 工作流程

```
1. 产品愿景 (58)
   │
   ├─→ 确定产品方向和长期目标
   │
   ├─→ 产品策略 (57)
   │   │
   │   ├─→ 定义战略方向和优先级
   │   │
   │   ├─→ 商业模式 (51)
   │   │   │
   │   │   └─→ 精益画布 (52) 或 创业画布 (59)
   │   │       │
   │   │       └─→ 可视化商业模式
   │   │
   │   ├─→ 安索夫矩阵 (50)
   │   │   │
   │   │   └─→ 选择增长策略路径
   │   │
   │   ├─→ PESTLE分析 (54)
   │   │   │
   │   │   └─→ 评估宏观环境因素
   │   │
   │   ├─→ 波特五力 (55)
   │   │   │
   │   │   └─→ 分析行业竞争力
   │   │
   │   └─→ 价值主张 (61)
   │       │
   │       └─→ 定义产品价值定位
   │
   └─→ 进入市场研究阶段
```

### 💡 关键决策点
- 产品愿景是否清晰？
- 商业模式是否可行？
- 增长策略路径是否正确？
- 价值主张是否具有差异化？

## 阶段二：市场研究

### 🎯 目标
深入了解市场、竞争对手和目标客户，为产品决策提供数据支持。

### 📋 使用技能

| 技能编号 | 技能名称 | 核心输出 | 使用时机 |
|---------|----------|----------|---------|
| **28** | 市场规模估算 | TAM/SAM/SOM估算 | 市场机会评估、投资决策 |
| **25** | 竞争对手分析 | 竞争对手优劣势和差异化 | 竞争对手研究、市场定位 |
| **27** | 市场细分 | 市场细分群体 | 目标市场定义、产品定位 |
| **24** | 理想客户画像 | 理想客户画像(ICP) | 目标市场定义、销售对齐 |
| **30** | 用户画像 | 详细用户画像 | 用户研究、产品设计、营销定位 |
| **31** | 用户细分 | 用户群体细分 | 个性化策略、营销定位 |
| **26** | 客户旅程地图 | 客户完整旅程映射 | 用户体验设计、痛点识别 |
| **29** | 情感分析 | 客户反馈和情感分析 | 客户反馈分析、品牌监控 |

### 🔄 工作流程

```
战略规划阶段
   │
   └─→ 市场规模估算 (28)
       │
       ├─→ 评估市场机会大小
       │
       ├─→ 市场细分 (27)
       │   │
       │   ├─→ 划分市场群体
       │   │
       │   ├─→ 理想客户画像 (24)
       │   │   │
       │   │   ├─→ 定义目标客户
       │   │   │
       │   │   └─→ 用户画像 (30)
       │   │       │
       │   │       └─→ 创建详细用户画像
       │   │
       │   ├─→ 用户细分 (31)
       │   │   │
       │   │   └─→ 细分用户群体
       │   │
       │   ├─→ 竞争对手分析 (25)
       │   │   │
       │   │   ├─→ 识别5个主要竞争对手
       │   │   │
       │   │   └─→ 分析优劣势和差异化
       │   │
       │   ├─→ 客户旅程地图 (26)
       │   │   │
       │   │   └─→ 映射客户完整旅程
       │   │
       │   └─→ 情感分析 (29)
       │       │
       │       └─→ 分析客户反馈和情感
       │
       └─→ 进入产品发现阶段
```

### 💡 关键决策点
- 市场规模是否足够大？
- 目标客户是否清晰？
- 竞争对手有哪些优势和劣势？
- 客户旅程中存在哪些痛点？

## 阶段五：数据分析

### 🎯 目标
通过数据驱动决策，验证假设，优化产品性能。

### 📋 使用技能

| 技能编号 | 技能名称 | 核心输出 | 使用时机 |
|---------|----------|----------|---------|
| **01** | A/B测试分析 | A/B测试结果分析 | 评估实验结果、检查显著性 |
| **02** | 队列分析 | 用户留存和参与模式分析 | 分析用户留存、研究功能采用 |
| **03** | SQL查询 | SQL查询编写和优化 | 数据提取、分析查询、数据管道 |

### 🔄 工作流程

```
产品执行阶段
   │
   └─→ SQL查询 (03)
       │
       ├─→ 提取和分析数据
       │
       ├─→ 队列分析 (02)
       │   │
       │   ├─→ 分析用户留存模式
       │   │
       │   ├─→ 识别功能采用趋势
       │   │
       │   └─→ 发现流失模式
       │
       └─→ A/B测试分析 (01)
           │
           ├─→ 验证实验结果
           │
           ├─→ 检查统计显著性
           │
           ├─→ 分析护栏指标
           │
           └─→ 提供发布/扩展/停止建议
           │
           └─→ 返回产品发现或执行阶段
```

### 💡 关键决策点
- 实验是否达到统计显著性？
- 用户留存趋势如何？
- 是否需要更多数据？
- 功能是否应该发布？

## 总结

这个用户旅程展示了产品经理如何：

1. **从战略规划开始**（8个技能）- 定义愿景、策略和商业模式
2. **进行市场研究**（7个技能）- 了解市场、竞争和客户
3. **发现产品机会**（13个技能）- 验证假设、生成创意、设计实验
4. **执行产品开发**（15个技能）- 编写PRD、规划冲刺、测试功能
5. **分析数据洞察**（3个技能）- 验证假设、优化产品
6. **推广到市场**（6个技能）- 制定GTM策略、选择目标市场
7. **驱动营销增长**（5个技能）- 生成创意、优化定位、跟踪指标
8. **持续优化循环** - 使用复盘和学习技能不断改进

**工具箱技能**（5个）贯穿整个旅程，提供文档模板和支持。

通过遵循这个旅程，产品经理可以系统化地使用所有65个技能，覆盖产品管理的完整生命周期。
