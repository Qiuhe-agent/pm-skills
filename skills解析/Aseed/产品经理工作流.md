## 阶段1：战略规划

**目标：** 建立清晰的产品方向

| 活动 | 输出 |
|------|------|
| 产品定位声明 | 定位文档 |
| 市场分析（PESTEL/TAM） | 市场规模报告 |
| AI技术可行性评估 | 可行性报告 |
| 战略路线图 | 3年路线图 |

**检查点：**
- 定位是否差异化明确？
- 市场规模是否足够？
- AI方案是否可行？

---

## 阶段2：需求发现

**目标：** 理解用户痛点，识别AI价值

| 活动 | 输出 |
|------|------|
| 用户角色画像 | 3-5个核心角色 |
| 问题陈述 | 痛点文档 |
| JTBD分析 | 待办任务分析 |
| AI问题框架 | AI适配判断 |

**检查点：**
- 是真痛点还是伪需求？
- AI是最佳方案吗？

---

## 阶段3：需求规划

**目标：** 筛选高价值需求

**优先级评估（5维）：**
- 用户价值 30%
- 业务价值 25%
- 技术可行性 20%
- 成本投入 15%
- 风险可控 10%

| 活动 | 输出 |
|------|------|
| 功能优先级矩阵 | Must-Have/Should-Have |
| MVP范围定义 | MVP说明书 |
| 迭代规划 | Sprint计划 |

**检查点：**
- MVP是否聚焦核心价值？

---

## 阶段4：PRD编写

**目标：** 清晰可执行的需求文档

**PRD结构：**
1. 产品概览（愿景、定位、目标）
2. 用户角色
3. 功能需求（用户故事+验收标准）
4. 非功能需求（性能、安全等）
5. AI特有需求（可解释性、置信度、人工兜底）
6. 发布计划
7. 成功指标

**检查点：**
- 验收标准可测试吗？
- AI有明确的人工兜底吗？

---

## 阶段5：设计验证

**目标：** 友好的设计方案

**AI设计要点：**
- 可解释性：展示决策依据
- 人工干预：允许修正AI建议
- 反馈闭环：收集用户反馈

| 活动 | 输出 |
|------|------|
| 交互/视觉设计 | 设计稿 |
| AI决策可视化 | AI界面设计 |
| 原型验证 | 测试报告 |

---

## 阶段6：开发发布

**目标：** 高效交付，质量保障

**Scrum流程：**
- Sprint规划会 → 每日站会 → Sprint评审 → Sprint回顾

**AI开发注意：**
- 数据准备（训练/验证/测试）
- 模型训练与验证
- 线上效果监控
- 小步快跑、频繁验证

**发布流程：**
发布计划 → 灰度发布（1-2周） → 全量发布

## 阶段7：增长优化

**目标：** 数据驱动迭代

### AARRR 海盗模型（增长漏斗）

由 Dave McClure 提出，5个环节形似海盗的钩子：

```
获客 Acquisition
    ↓
激活 Activation
    ↓
留存 Retention
    ↓
变现 Revenue
    ↓
推荐 Referral
```

| 环节 | 目标 | 核心指标 | 关键动作 |
|------|------|----------|----------|
| **获客** | 获取用户 | 下载量、注册量、CAC | 渠道优化、SEO/ASO |
| **激活** | 体验到价值 | 激活率、Aha!时间 | 引导优化、新手教程 |
| **留存** | 持续使用 | 次日/7日/30日留存 | 习惯养成、召回策略 |
| **变现** | 获得收入 | ARPU、付费率、LTV | 定价策略、增值服务 |
| **推荐** | 用户传播 | NPS、推荐率、K因子 | 邀请奖励、社交分享 |

### 核心指标体系

| 类别 | 示例 |
|------|------|
| 用户获取 | DAU/MAU、CAC |
| 用户激活 | 激活率、核心功能使用 |
| 用户留存 | 次日/7日/30日留存 |
| 用户变现 | ARPU、付费率 |
| AI效果 | 准确率、自动化率 |
| 业务结果 | NPS、目标达成率 |

| 活动 | 输出 |
|------|------|
| 数据埋点 | 埋点方案 |
| 业务健康诊断 | 月度/季度报告 |
| A/B测试 | 测试报告 |
| 迭代优化 | 优化计划 |

## 阶段8：生命周期管理

**目标：** 最大化产品价值

**生命周期4阶段：**

| 阶段 | 特征 | 重点 |
|------|------|------|
| 引入期 | 验证PMF | 快速迭代、标杆客户 |
| 成长期 | 用户增长 | 功能完善、生态建设 |
| 成熟期 | 收入增长 | 成本控制、客户成功 |
| 衰退期 | 用户流失 | 转型探索、有序退市 |

---

## 阶段9：团队协作

**目标：** 高效协作机制

**AI团队关键角色：**
- AI产品经理：产品定义、跨团队协调
- AI科学家：算法研究、模型开发
- AI工程师：模型工程化、数据管道
- 数据工程师：数据采集、数据平台
- 产品设计师：用户体验、界面设计

**关键会议：**
- 需求澄清会（每迭代）
- 干系人沟通（每周/每月）

---

## 阶段10：职业发展

**目标：** 持续成长

**能力模型：**
- 产品基础：用户研究、需求分析、数据分析
- AI理解：ML基础、AI伦理、模型评估
- 商业思维：市场分析、增长思维、战略思考
- 技术协作：技术沟通、项目管理
- 领导力：愿景驱动、影响力、人才培养

**发展路径：**
初级 → 中级（1-2年）→ 高级（2-3年）→ 专家/负责人（3-5年）→ CPO（5年+）

**关键动作：**
- 能力自评（每半年）
- 发展规划（每年）
- 复盘反思（每项目/每季度）

## 核心原则

1. **用户价值核心** — 始终以解决用户问题为出发点
2. **数据驱动决策** — 让数据和反馈指导迭代
3. **小步快跑验证** — MVP、灰度、A/B测试
4. **AI+人双轮驱动** — AI可解释、人工可兜底
5. **流程是工具不是目的** — 根据实际灵活调整
