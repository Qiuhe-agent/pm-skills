## 1. 项目背景

### 1.1 产品概述

Coze是一个**多端零代码AI Bot开发平台**，让用户可以快速创建、部署和分享AI助手，无需编写代码。

### 1.2 目标用户

1. **内容创作者/社群运营** - UP主、博主、群主
2. **中小企业运营者** - 电商、教育、本地生活
3. **独立开发者/创业者** - MVP验证、效率工具
4. **企业数字化团队** - 内部工具、Bot中台

### 1.3 业务目标

| 时间 | 目标 |
|------|------|
| **Q1 2026** | 月活创作者10,000+，Bot创建数100,000+ |
| **Q2 2026** | 月活创作者50,000+，插件数500+ |

## 4. 功能详情

### 4.1 Bot创建模块

#### 4.1.1 可视化编排

**功能描述**：
通过拖拽方式编排Bot的工作流，无需编写代码。

**用户流程**：
1. 进入Bot编辑页面
2. 从左侧组件库拖拽节点到画布
3. 连接节点，定义流程
4. 配置节点参数
5. 测试运行
6. 保存发布

**功能要点**：
- 支持的节点类型：
  - LLM节点（对话、生成）
  - 知识库节点（检索、引用）
  - 条件节点（if/else、switch）
  - 循环节点（for、while）
  - 插件节点（第三方插件）
  - 变量节点（设置、获取）
  - 结束节点
- 画布操作：
  - 拖拽移动
  - 缩放（0.5x - 2x）
  - 撤销/重做
  - 框选、复制、粘贴
- 自动布局：
  - 自动对齐
  - 智能连线
  - 美观的布局

**非功能需求**：
- 画布流畅度：60fps
- 支持节点数：最多100个
- 保存响应时间：<2秒

#### 4.1.2 模板库

**功能描述**：
提供各行业、各场景的Bot模板，用户可以一键使用。

**模板分类**：
- 按行业：电商、教育、金融、医疗等
- 按场景：客服、营销、社群、助手等
- 按角色：AI助手、顾问、教练、客服等

**模板使用流程**：
1. 浏览模板库
2. 查看模板详情和演示
3. 点击"使用此模板"
4. 自定义配置（修改提示词、变量等）
5. 测试运行
6. 发布

**模板要求**：
- 模板数量：v1.0提供100+
- 模板质量：每个模板都经过测试
- 模板更新：每月更新20+新模板

### 4.2 多端发布模块

#### 4.2.1 发布平台支持

**v1.0支持的平台**：
1. 微信公众号
2. 微信小程序
3. 企业微信
4. 飞书
5. 钉钉
6. Discord
7. Slack
8. Telegram
9. Web（网页）
10. API

**v1.1计划支持**：
11. 抖音
12. 快手
13. 小红书
14. B站
15. WhatsApp

**平台配置**：
每个平台需要配置的信息：
- 微信公众号：AppID、AppSecret、Token
- 飞书：App ID、App Secret
- Discord：Bot Token
- Web：嵌入代码
- API：API Key

#### 4.2.2 一键发布

**功能描述**：
一次配置，一键发布到多个平台。

**用户流程**：
1. 在Bot编辑页面点击"发布"
2. 选择要发布的平台（可多选）
3. 配置各平台的参数
4. 点击"一键发布"
5. 查看发布状态
6. 发布成功后获取访问链接

**发布状态**：
- 待发布
- 发布中
- 发布成功
- 发布失败（显示错误原因）

### 4.3 知识库模块

#### 4.3.1 文档上传

**支持的格式**：
- PDF
- Word（.doc, .docx）
- PowerPoint（.ppt, .pptx）
- Excel（.xls, .xlsx）
- TXT
- Markdown
- HTML
- 网页URL
- 图片（OCR识别）

**上传流程**：
1. 进入知识库页面
2. 点击"上传文档"或拖拽文件
3. 选择文件或输入URL
4. 等待处理（解析、向量化）
5. 处理完成，查看文档

**处理限制**：
- 免费版：单个文件<10MB，总共<100MB
- Pro版：单个文件<100MB，总共<10GB
- 企业版：无限制

#### 4.3.2 检索增强（RAG）

**功能描述**：
Bot在回答问题时，先从知识库检索相关内容，再基于检索结果回答。

**检索策略**：
- 向量检索（语义相似）
- 关键词检索（精确匹配）
- 混合检索（向量+关键词）

**RAG配置**：
- 检索结果数量：1-10条
- 相似度阈值：0.5-0.9
- 引用显示：开启/关闭
- 未找到时的处理：直接回答 / 拒绝回答
