# Coze AI用户访谈脚本和洞察报告

## 文档概述

本文档提供Coze用户访谈的完整脚本框架，以及访谈后的洞察分析模板，帮助产品团队更好地理解用户需求。

---

## 一、访谈准备

### 1.1 访谈目标

| 目标 | 说明 |
|------|------|
| **验证痛点** | 用户是否真的有我们假设的痛点？ |
| **了解需求** | 用户希望如何解决这些问题？ |
| **收集反馈** | 用户对我们产品的看法？ |
| **发现机会** | 有哪些我们没想到的需求？ |

### 1.2 受访者筛选

**受访者画像**：
- 个人创作者：UP主、博主、群主
- 中小企业：电商、教育、本地生活
- 企业数字化：IT、数字化团队
- 技术极客：开发者、创业者

**招募渠道**：
- 产品内招募
- 社群招募
- 朋友推荐
- 付费招募

**激励措施**：
- ¥100-200 红包
- Pro版会员
- 优先体验新功能

## 二、访谈脚本（60分钟）

### 第一部分：热身与背景（10分钟）

**[1] 开场与介绍**
```
你好！我是Coze的产品经理XXX。

非常感谢你抽出时间来参加今天的访谈！

今天我们想了解一下你在Bot开发和使用方面的经验和想法。

我们的访谈大约需要60分钟，我们会录音，但只会用于内部分析，不会对外公开。

你有什么问题吗？

好的，那我们开始吧！
```

**[2] 简单自我介绍**
```
首先，能简单介绍一下你自己吗？
- 你是做什么工作的？
- 你的主要职责是什么？
```

**[3] 你当前的工作/生活状态**
```
能聊聊你平时的一天大概是什么样的吗？
- 最忙的是什么时候？
- 最让你头疼的是什么事？
```

---

### 第二部分：痛点深挖（20分钟）

**[4] 你遇到过什么问题？**
```
在你的工作/生活中，有没有遇到过这些情况？
- 需要重复回答很多类似的问题？
- 粉丝/客户太多，回复不过来？
- 想做个Bot，但不会写代码？
- 在多个平台都有粉丝，分别管理很麻烦？

[如果有] 能详细说说吗？
- 这个问题多久发生一次？
- 对你影响有多大？
- 你现在是怎么解决的？
```

**[5] 当前的解决方案**
```
对于这些问题，你现在是怎么解决的？
- 用了什么工具？
- 效果怎么样？
- 有什么不满意的地方？
```

**[6] 理想的解决方案**
```
如果有一个完美的解决方案，你希望它是什么样的？
- 它能帮你做什么？
- 你希望它怎么工作？
- 有什么功能是必须有的？
```

---

### 第三部分：产品反馈（20分钟）

**[7] 你是怎么知道Coze的？**
```
你是怎么知道Coze的？
- 朋友推荐？
- 刷到内容？
- 搜索找到的？
- 其他？
```

**[8] 使用体验如何？**
```
能说说你用Coze的体验吗？
- 最喜欢什么？
- 最不喜欢什么？
- 如果能改进一个地方，是什么？
```

**[9] 具体功能反馈**
```
我们来聊聊几个具体功能：

[新手引导]
- 新手引导怎么样？
- 能快速上手吗？
- 有什么困惑的地方？

[Bot创建]
- 创建Bot容易吗？
- 模板够用吗？
- 有什么难点？

[多端发布]
- 发布到多个平台方便吗？
- 还需要支持哪些平台？

[知识库]
- 知识库好用吗？
- 回答准确吗？
- 有什么改进建议？

[插件]
- 插件够用吗？
- 还需要什么插件？
```

**[10] 付费意愿**
```
如果Coze有Pro版，你愿意付费吗？
- 为什么愿意/不愿意？
- 你觉得多少钱合适？
- 什么功能会让你愿意付费？
```

---

### 第四部分：总结与感谢（10分钟）

**[11] 还有什么想聊的吗？**
```
我们准备的问题差不多了，你还有什么想和我们聊的吗？
- 有什么我们没问到，但你觉得很重要的？
- 有什么建议给我们？
```

**[12] 总结与感谢**
```
非常感谢你今天的分享！
对我们太有帮助了！

我们会认真考虑你的建议。

[如果有后续测试] 后续我们有新功能，想请你先试试，愿意吗？

再次感谢！
我们的小礼物会稍后发给你。

保持联系！
```

## 六、访谈最佳实践

### 6.1 访谈技巧

**做的好的**：
- 多听少说，让用户多说
- 追问"为什么"，深挖痛点
- 不引导，不推销产品
- 记录原话，不加工
- 保持中立，不评判

**避免做的**：
- 不要打断用户
- 不要推销产品
- 不要引导用户说你想听的
- 不要争辩
- 不要急于给出解决方案

### 6.2 访谈后续

**24小时内**：
- 整理访谈记录
- 标记关键要点
- 发送感谢邮件

**48小时内**：
- 团队内部分享
- 初步洞察汇总
- 更新假设

**1周内**：
- 完整洞察报告
- 优先级排序
- 行动计划
