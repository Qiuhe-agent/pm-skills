# Coze AI实验设计文档

## 文档概述

本文档定义Coze产品实验的设计框架、模板、最佳实践，帮助产品团队科学地进行实验，降低决策风险。

## 一、实验框架

### 1.1 实验类型

| 类型 | 说明 | 适用场景 |
|------|------|---------|
| **A/B测试** | 两个版本对比 | 功能优化、UI改进 |
| **多变量测试** | 多个变量组合 | 复杂页面优化 |
| **MVP测试** | 最小可行产品 | 新功能验证 |
| **伪门测试** | 看点击不实现 | 需求验证 |
| **用户分组** | 不同用户组对比 | 留存、增长实验 |

## 三、实验示例

### 实验E-001：新手引导简化

**实验假设**：
> 如果我们将新手引导从10步减少到5步，并增加"一键创建"模板，那么新用户7天激活率会从30%提升到40%（提升33%），因为用户能更快获得成功体验。

**实验背景**：
- 当前新用户7天激活率只有30%
- 用户反馈"新手引导太复杂"、"不知道从哪开始"
- 竞品的新手引导更简单

**实验设计**：
- **对照组（A）**：当前10步新手引导
- **实验组（B）**：新5步引导 + 一键创建模板

**实验变量**：
1. 引导步骤数：10步 vs 5步
2. 一键创建模板：无 vs 有

**目标用户**：
- 受众：所有新注册用户
- 分流比例：A组50%，B组50%
- 样本量：每组5,000用户（共10,000用户）

**实验周期**：
- 开始日期：2026-03-15
- 结束日期：2026-03-29
- 预计时长：14天

**衡量指标**：

主要指标：
- 7天激活率：完成首个Bot创建并发布
- 目标：从30%提升到40%

次要指标：
- 新手引导完成率：从55%提升到70%
- 首个Bot创建时间：从60分钟降到30分钟
- 次日留存率：从40%提升到45%

护栏指标：
- 7天留存率：不低于当前水平（±2%）

**成功标准**：
- 主要指标提升33%（30%→40%）
- 统计显著性p < 0.05
- 护栏指标下降不超过2%

**实验分组**：
- 分组方法：完全随机
- 分流键：user_id
- 缓存策略：稳定（同一用户始终在同一组）

**数据分析计划**：
- 分析工具：内部数据平台 + Amplitude
- 统计方法：卡方检验（激活率）、T检验（创建时间）
- 中期检查：第7天（如果效果明显，可提前结束）

**风险与预案**：
- 风险1：B组用户困惑，激活率反而下降 → 预案：准备回滚开关
- 风险2：样本量不够，统计不显著 → 预案：延长实验周期

**后续计划**：
- 如果成功：全量发布，继续优化后续步骤
- 如果失败：回滚，分析原因，重新设计
- 如果不确定：延长实验，收集更多数据

### 实验E-002：Pro版定价测试

**实验假设**：
> 如果我们提供¥9.9/月的入门版Pro，那么免费→付费转化率会从3%提升到6%（翻倍），因为更低的价格降低了决策门槛。

**实验背景**：
- 当前免费→付费转化率只有3%
- 用户反馈"Pro版有点贵"
- 需要验证价格敏感度

**实验设计**：
- **对照组（A）**：Pro版 ¥29/月
- **实验组（B1）**：Pro版 ¥19/月
- **实验组（B2）**：入门版 ¥9.9/月 + Pro版 ¥29/月

**目标用户**：
- 受众：免费版使用7天以上的活跃用户
- 分流比例：A组34%，B1组33%，B2组33%
- 样本量：每组3,000用户（共9,000用户）

**实验周期**：21天

**衡量指标**：

主要指标：
- 免费→付费转化率：从3%提升到6%

次要指标：
- ARPU（每用户平均收入）
- 付费用户7天留存率
- 退款率

护栏指标：
- 用户满意度NPS

## 四、样本量计算

### 4.1 样本量公式

对于二项分布指标（如转化率）：

```
n = (Zα/2 + Zβ)² × p × (1-p) / Δ²
```

其中：
- Zα/2 = 1.96（95%置信度）
- Zβ = 0.84（80%统计功效）
- p = 基准转化率
- Δ = 期望提升的绝对值

### 4.2 样本量计算器

| 基准转化率 | 期望提升 | 每组样本量 |
|-----------|---------|-----------|
| 30% | +10%（到40%） | 380 |
| 30% | +5%（到35%） | 1,300 |
| 5% | +3%（到8%） | 1,000 |
| 5% | +2%（到7%） | 2,300 |
| 2% | +1%（到3%） | 3,100 |

**经验法则**：
- 每组至少1,000用户
- 如果可能，每组5,000用户
- 小流量（1-5%）先测，没问题再放大

## 五、统计显著性

### 5.1 p值解释

| p值 | 解释 | 结论 |
|------|------|------|
| p < 0.01 | 非常显著 | ✅ 强烈建议发布 |
| p < 0.05 | 统计显著 | ✅ 可以发布 |
| p < 0.10 | 边缘显著 | 🟡 收集更多数据 |
| p > 0.10 | 不显著 | ❌ 不发布或重新设计 |

### 5.2 置信区间

```
效果 = 实验组转化率 - 对照组转化率
置信区间 = 效果 ± 1.96 × 标准误差
```

**示例**：
- 对照组转化率：30%
- 实验组转化率：40%
- 效果：+10%
- 置信区间：+6% ~ +14%
- 结论：✅ 效果显著，因为置信区间不包含0

## 七、实验最佳实践

### 7.1 DO's

✅ **一次只测试一个变量** - 否则不知道是什么导致的变化

✅ **提前计算样本量** - 避免数据不足

✅ **使用护栏指标** - 确保没有负面影响

✅ **设定成功标准** - 提前定义什么算成功

✅ **随机分组** - 避免偏差

✅ **稳定分组** - 同一用户始终在同一组

✅ **中期检查** - 但不要频繁看数据

✅ **完整周期** - 至少包含一个周末

### 7.2 DON'Ts

❌ **不要偷看数据** - 频繁检查会增加假阳性

❌ **不要提前结束** - 除非效果非常明显

❌ **不要忽略护栏指标** - 主要指标好但其他指标差也不行

❌ **不要过度解读** - 统计显著≠实际重要

❌ **不要一次测试太多** - 聚焦最重要的假设
