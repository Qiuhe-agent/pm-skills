
# 知识城邦 (Knowledge Citadel) - 完整策划方案

## 一、项目概述

### 1.1 项目愿景
将个人知识管理转化为一场引人入胜的城市建设游戏。每一条知识都是一块砖瓦，每一个领域都是一个城区，用户在积累知识的同时，见证自己的知识城邦从一个小村庄逐步发展成繁荣的知识帝国。

### 1.2 核心概念
- **知识资源 = 建筑材料**：用户提供的学习资源是建造的原材料
- **学习计划 = 建筑蓝图**：大模型制定的学习计划是施工蓝图
- **学习过程 = 建造过程**：每完成一个学习步骤，建筑就建好了一部分
- **知识掌握 = 建筑完工**：完全掌握时，建筑落成并点亮
- **领域 = 城区**：不同知识分类对应城邦的不同功能区域
- **关联 = 道路/桥梁**：知识间的连接构建城邦的交通网络
- **熟练度 = 建筑等级**：知识掌握程度决定建筑高度和华丽程度

## 三、新增：学习计划与建筑建造系统

### 3.3 资源输入与学习计划制定

```
用户输入学习资源
    ↓
大模型分析资源内容
    ↓
评估学习难度和时长
    ↓
生成分阶段学习计划
    ↓
生成建筑蓝图（带建造进度）
    ↓
在城邦中放置建筑工地
    ↓
开始学习建造过程
```

### 3.4 建筑建造的五个阶段

| 阶段 | 建造进度 | 视觉表现 | 学习活动 |
|-----|---------|---------|---------|
| 🏗️ 奠基 | 0-20% | 建筑地基 + 脚手架 | 了解基础概念、阅读资源 |
| 🔨 搭建 | 20-50% | 墙体框架 + 屋顶雏形 | 实践练习、完成小任务 |
| 🏠 封顶 | 50-80% | 建筑主体完成 + 外墙 | 综合应用、项目实战 |
| 🎨 装修 | 80-100% | 内部装饰 + 灯光效果 | 深入理解、融会贯通 |
| ✨ 落成 | 100% | 完整建筑 + 点亮特效 | 知识掌握、建筑完工 |

### 3.5 学习资源输入方式

**支持的资源类型：**

1. **文本资源**
   - 网页链接
   - 文章段落
   - 笔记内容
   - 书籍章节

2. **课程资源**
   - 在线课程链接
   - 视频教程列表
   - 课程大纲

3. **实践项目**
   - 项目描述
   - 实践任务
   - 编码挑战

4. **自定义学习**
   - 用户描述学习目标
   - 大模型生成学习路径

## 八、完整工作流程

### 8.1 用户使用流程

```
1. 初始化城邦
   └─→ 生成初始城市状态文件
   └─→ 创建基础HTML模板

2. 开始学习 (核心流程)
   └─→ 用户输入学习资源/目标
        例："我想学React，这是教程链接"
   └─→ Claude Code分析资源
        ├─→ 调用资源分析提示词
        ├─→ 评估难度和时长
        └─→ 生成学习计划
   └─→ 创建建筑工地
        ├─→ 确定建筑类型和位置
        └─→ 显示奠基阶段的建筑
   └─→ 开始分阶段学习

3. 学习进行中
   └─→ 完成当前任务 → 报告进度
   └─→ Claude Code检验理解
        ├─→ 生成检验问题
        ├─→ 用户回答验证
        └─→ 更新进度
   └─→ 更新建筑视觉
        ├─→ 建筑建造状态更新
        └─→ 生成新的HTML
```

## 十五、风险与应对

| 风险 | 影响 | 应对策略 |
|-----|------|---------|
| 大模型输出不稳定 | 高 | 提供明确的输出格式要求，多轮验证 |
| 用户知识输入不规范 | 中 | 提供输入引导，示例提示 |
| 生成HTML过大 | 中 | 分批生成，渐进式更新 |
| 用户热情衰减 | 高 | 成就系统激励，定期反馈 |

## 结语

知识城邦 2.0 的核心理念是：**让学习过程可视化，让每一点进步都有见证。

你不再是"添加"知识，而是"建造"知识；
每一步学习都对应建筑的一砖一瓦；
学习的过程，就是看着自己的城邦从无到有、从小到大的过程。

**学习资源 → 学习计划 → 建筑工地 → 分阶段建造 → 建筑落成 → 点亮城邦**

这不仅仅是知识管理，这是一场关于成长的建造游戏。
