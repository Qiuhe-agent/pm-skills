# 方案二：Minecraft 隐喻优先架构

## 核心理念

完全围绕 Minecraft 概念设计。将知识管理系统的每一个部分都直接映射到 Minecraft 的游戏概念上，让整个系统充满游戏感和想象力。

## 核心映射

```
知识条目 → Minecraft 方块/物品
知识分类 → Minecraft 生物群系/建筑类型
知识关联 → 建筑之间的道路/桥梁
时间生长 → 建筑的建造阶段（地基→框架→完工）
领域 → Minecraft 区域/村庄
```

## 详细映射设计

### 1. 世界与区域

| 知识概念 | Minecraft 概念 | 说明 |
|---------|---------------|------|
| 知识领域 | 生物群系 | 森林=自然科学，沙漠=商业，海洋=文学... |
| 知识主题 | 村庄/定居点 | 每个村庄聚焦一个主题 |
| 个人空间 | 主基地/家 | 用户的核心区域 |
| 存档 | 世界 | 整个知识城邦是一个 Minecraft 世界 |

### 2. 建筑系统

| 知识概念 | Minecraft 概念 | 说明 |
|---------|---------------|------|
| 笔记/文章 | 图书馆 | 存放文字内容 |
| 项目/任务 | 工厂/工坊 | 生产性内容 |
| 灵感/想法 | 花园/温室 | 创意内容 |
| 参考资料 | 仓库/储藏室 | 收集的资料 |
| 里程碑 | 纪念碑/雕像 | 重要节点 |

**建筑类型库**：
- 图书馆（笔记）：木质结构，多层，书架
- 工厂（项目）：石质结构，流水线感
- 花园（灵感）：彩色玻璃，植物，喷泉
- 仓库（资料）：石头，箱子，货架
- 纪念碑（里程碑）：石英，高塔，雕像

### 3. 方块与物品

| 知识属性 | Minecraft 方块/物品 | 说明 |
|---------|---------------------|------|
| 文本内容 | 书架/成书 | 每本书代表一条笔记 |
| 链接 | 信标/传送门 | 连接到其他位置 |
| 图片/附件 | 展示框/物品展示 | 可视化内容 |
| 标签 | 旗帜/图案 | 不同图案代表不同标签 |
| 重要性 | 方块类型 | 钻石块>金块>铁块>石块 |

### 4. 时间生长

建造阶段对应知识积累过程：

| 阶段 | Minecraft 表现 | 知识状态 |
|-----|---------------|---------|
| 1. 选址 | 清理地形，打地基 | 刚开始接触这个主题 |
| 2. 框架 | 主要结构，墙体 | 有了基本框架，关键点 |
| 3. 内部 | 家具，装饰，细节 | 内容充实，细节丰富 |
| 4. 完工 | 照明，美化，花园 | 完整的知识体系 |
| 5. 扩建 | 加层，翼楼，附属建筑 | 知识延伸，相关主题 |

### 5. 关联与交通

| 关联强度 | Minecraft 表现 |
|---------|---------------|
| 强关联 | 宽阔的石砖路，带路灯 |
| 中关联 | 泥土路/沙砾路 |
| 弱关联 | 小径，脚印 |
| 双向关联 | 双向道路，有来有回 |
| 核心节点 | 道路交汇，广场 |

## 数据层设计

### Minecraft 存档结构

```
知识城邦/
└── minecraft-world/
    ├── level.dat          # 世界元数据
    ├── icon.png           # 世界图标
    ├── data/              # 游戏数据
    │   └── villages.dat   # 村庄信息
    ├── region/            # 区块数据 (.mca)
    │   ├── r.0.0.mca
    │   ├── r.0.1.mca
    │   └── ...
    └── playerdata/        # 玩家数据
```

### 知识数据映射

每条知识对应：
- 一个方块或物品（在世界中的位置）
- NBT 标签存储元数据
- 成书（Book & Quill）存储内容

## HTML 可视化层

### 设计理念

做一个"简化版 Minecraft 浏览器"：
- 使用 Minecraft 的材质包
- 但简化渲染逻辑
- 保留核心游戏感

### 视图模式

#### 2D 地图视图
- 类似 Minecraft 小地图
- 从上往下看
- 不同颜色代表不同生物群系
- 图标代表建筑

#### 3D 简化视图
- Three.js + Minecraft 材质
- 可旋转、缩放
- 可走进建筑
- 简化物理（不需要碰撞检测）

#### 时间机器
- 滑块控制时间
- 看建筑从地基到完工的过程
- 关键节点高亮

---

## 优点

- 隐喻一致，想象力强
- Minecraft 玩家很容易理解
- 游戏化元素强，有动力积累知识
- 可以直接导出 Minecraft 存档
- 概念完整，故事性强

## 缺点

- 可能被 Minecraft 的概念限制住
- 如果后期想调整隐喻，成本较高
- 初期需要设计大量映射规则
- Minecraft 存档格式复杂
