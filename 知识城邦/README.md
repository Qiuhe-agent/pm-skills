# 知识城邦

管理个人碎片化信息，打造个人知识库，最终创建个人数字孪生。

## 项目愿景

```
碎片化信息管理 → 个人知识库 → 个人数字孪生
```

## 核心概念

- **空间维度**：用建筑展示领域知识
- **时间维度**：建筑随时间"生长"，不同日期展示不同形态
- **风格参考**：Minecraft
- **交互方式**：对话式存储，HTML 可视化展示

## 设计方案

### [方案一：分层渐进式架构](docs/方案一：分层渐进式架构.md) ⭐（推荐）

清晰分层，每一层可以独立演进。

```
对话交互层 → 知识处理层 → 城邦生成层 → 可视化层 → 数据层
```

**优点**：架构清晰、易于演进、适合 Skill + Subagent 模式

---

### [方案二：Minecraft 隐喻优先架构](docs/方案二：Minecraft隐喻优先架构.md)

完全围绕 Minecraft 概念设计。

```
知识 → 方块/物品，分类 → 生物群系，关联 → 道路，时间 → 建造阶段
```

**优点**：隐喻一致、游戏感强、可直接导出 Minecraft 存档

---

### [方案三：极简原型架构](docs/方案三：极简原型架构.md) 🏃（快速开始）

先做能跑的最小版本，快速验证概念。

**优点**：快速可用、易于调整、风险低

---

## 推荐路线

1. **先用方案三做原型**：快速验证概念
2. **再迁移到方案一**：架构化、可扩展
3. **酌情加入方案二元素**：Minecraft 风格和导出功能
